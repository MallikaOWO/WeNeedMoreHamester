# 同层前端卡开发流程

基于 tavern_helper_template 模板和同层前端教程(discord@poaries)整理。已针对本项目实际情况定制（React、自有分发、独立构建环境）。

---

## 核心概念

**同层前端卡** = 前端界面始终只显示最后一层楼，通过脚本隐藏历史楼层，所有交互通过酒馆助手函数完成（不触发楼层刷新），实现沉浸式连续体验。

---

## 最关键的能力：TypeScript/React → 单HTML文件打包

webpack 将结构化前端项目打包为**单个自包含 HTML 文件**，这是酒馆前端界面的最终消费格式。

### 打包原理

当 `index.ts`/`index.tsx`（入口）同目录下存在 `index.html`（模板），webpack 会：

1. **`HtmlWebpackPlugin`** — 以 index.html 为模板，注入打包后的 script 标签
2. **`HtmlInlineScriptWebpackPlugin`** — 将所有 JS 内联进 HTML（不产生外部 .js 文件）
3. **`MiniCssExtractPlugin` + `HTMLInlineCSSWebpackPlugin`** — 将所有 CSS 内联进 HTML
4. **`LimitChunkCountPlugin({ maxChunks: 1 })`** — 禁止代码拆分，强制单 chunk

最终输出：`dist/xxx/index.html`，一个包含全部 JS + CSS 的自包含文件，可直接用于酒馆前端界面配置。

### 入口判断逻辑

```
src/你的界面/
├── index.html   ← 存在此文件 → 走 HTML 内联模式，输出单 HTML
├── index.tsx    ← 入口脚本
└── ...

src/你的脚本/
├── index.ts     ← 不存在 index.html → 走纯脚本模式，输出 .js
└── ...
```

webpack 自动扫描 `src/**/index.{ts,tsx,js,jsx}`，无需手动配置入口。

### 最小 index.html 模板

```html
<!DOCTYPE html>
<html>
<head><meta charset="utf-8"></head>
<body>
  <div id="root"></div>
</body>
</html>
```

### 最小 index.tsx 入口（React）

```tsx
import React from 'react';
import { createRoot } from 'react-dom/client';
import App from './App';

$(() => {
  const container = document.getElementById('root');
  if (container) {
    createRoot(container).render(<App />);
  }
});
```

`$(() => { ... })` 是 jQuery 的 DOM ready，酒馆环境全局可用。

### externals 策略

webpack 对外部依赖的处理（已在 `webpack.config.ts` 中配置）：
- **本地路径/相对路径/http 开头/@别名** → 正常打包
- **`react`/`react-dom` 相关** → 正常打包进 HTML（不 external）
- **`jquery`/`lodash`/`yaml`/`zod`** 等 → 映射为酒馆全局变量（`$`、`_`等）
- **其他 npm 包** → 自动转为 jsdelivr CDN ESM import

## 开发工作流

```bash
# 在 WeNeedMoreHamester 目录下
pnpm install                # 安装依赖
pnpm watch                  # 开发模式：watch + schema + tavern_sync + 热重载 → dist/0.0.0/
pnpm build:dev              # 开发模式单次构建 → dist/0.0.0/
pnpm build                  # 生产模式构建 → dist/0.0.0/

# 发行版本（版本号使用标准 semver: 主版本.次版本.修订号）
npx webpack --mode production --env version=0.1.0    # → dist/0.1.0/
npx webpack --mode production --env version=1.0.0    # → dist/1.0.0/

# 同步角色卡
pnpm sync                   # 从酒馆提取角色卡数据到本地 鼠鼠天堂/ 目录
pnpm sync bundle            # 将本地数据打包为可导入酒馆的角色卡文件

# schema
pnpm dump                   # 手动将 schema.ts → schema.json（watch时自动执行）
```

`pnpm watch` 会同时启动：
- webpack 监听 `src/` 变化 → 自动重新打包到 `dist/0.0.0/`
- schema 监听 → `schema.ts` 变化时自动生成 `schema.json`
- tavern_sync → 自动将 dist 产物同步到酒馆
- Socket.IO 服务（端口 6621）→ 通知酒馆助手热重载前端

详细命令参见 `WeNeedMoreHamester/命令速查.md`。

## 同层交互核心流程

玩家点击选项/发送消息时，**不走酒馆原生发送流程**，而是：

```
① 构建提示词（玩家输入/选项文本）
② createChatMessages({role:'user', message, data:mvu_data}, {refresh:'none'})
   → 静默创建user楼层，不刷新显示
③ generate({user_input, should_stream:true})
   → 调用LLM生成回复，可监听流式token
④ 过滤<thinking>标签，提取<maintext>、<option>、<sum>等自定义标签
⑤ Mvu.parseMessage(message, baseData) → 解析变量更新命令
⑥ createChatMessages({role:'assistant', message, data:parsedData}, {refresh:'none'})
   → 静默创建assistant楼层
⑦ 前端根据新楼层数据刷新显示
```

**错误回退**: generate失败或格式不合规范 → 删除已创建的user楼层 → 恢复到发送前状态

**信息过滤**: 忽略 `<thinking>`/`<think>` 标签内的所有内容，防止思维链干扰解析

## 隐藏楼层脚本

角色脚本，只保留最后一层楼的DOM显示：

```js
$(() => {
  $('#chat > .mes').not('.last_mes').remove();
  let current_chat_id = SillyTavern.getCurrentChatId();
  eventOn(tavern_events.CHAT_CHANGED, chat_id => {
    if (current_chat_id !== chat_id) {
      current_chat_id = chat_id;
      reloadIframe();
    }
  })
})
```

## 酒馆助手关键API速查

| 函数 | 用途 |
|------|------|
| `getChatMessages(range, {role?})` | 获取楼层消息 |
| `getLastMessageId()` | 获取最新楼层ID |
| `createChatMessages(msgs, {refresh})` | 创建楼层（refresh:'none' 不刷新显示） |
| `deleteChatMessages(ids, {refresh})` | 删除楼层 |
| `generate({user_input, should_stream})` | 调用LLM生成 |
| `getVariables({type, message_id})` | 读取楼层变量 |
| `updateVariablesWith(updater, {type, message_id})` | 原子更新变量 |
| `Mvu.getMvuData({type, message_id})` | 读取MVU数据（stat_data等） |
| `Mvu.parseMessage(message, base)` | 解析消息中的变量更新命令 |
| `waitGlobalInitialized('Mvu')` | 等待MVU初始化完成 |
| `eventOn(event, handler)` | 监听事件 |
| `reloadIframe()` | 重载前端界面 |
