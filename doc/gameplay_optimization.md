# 鼠鼠天堂 游戏性优化方案

## 一、当前问题诊断

### 1.1 资源经济失衡 — 无压力感

**现象：** 玩家很快进入"资源溢出"状态，能源和心情始终充足，缺乏有意义的资源决策。

**根因分析：**

| 问题 | 具体数值 | 影响 |
|------|----------|------|
| 成本曲线平坦 | 设施从 20→30→50→60→80→100，近似线性 | 攒几回合就够下一级，无需取舍 |
| 无维护成本 | 设施建成后零消耗 | 资源只增不减，中期开始堆积 |
| 产能增长快于需求 | 跑轮(10)+鼠鼠(5)×心情≈15/回合，而下一个设施才50 | 3-4回合攒够，无紧迫感 |
| 能源上限提升太便宜 | 向日葵储能站 40 能源，+100 上限 | 很快突破瓶颈 |
| 星尘来源充足 | 成就奖励总计 85 星尘，天使升级仅 10/25/50/100 | 前期成就覆盖大部分升级 |
| 体力恢复太快 | 休息+20/回合 vs 消耗15/回合 | 1回合满血，无轮换需求 |

### 1.2 天使系统缺乏深度

| 问题 | 详情 |
|------|------|
| 多个技能是 stub | emergency_repair、preview_event、force_event_type 未完整实现 |
| 技能效果不可感知 | boost_production 仅标记 "simplified"，无明确反馈 |
| 升级无紧迫性 | Lv.1 天使即可管理大部分设施，升级仅解锁高级建筑 |
| 领域匹配过于宽松 | function 类设施 manageDomain='any'，无需策略安排 |
| 泡芙（social）缺管理对象 | 仅瞭望台需要 social 域，实际用途有限 |

### 1.3 事件系统缺乏后果

| 问题 | 详情 |
|------|------|
| 选项效果单一 | 仅有 energy/stardust/mood delta，无持续效果 |
| 挑战事件无特殊机制 | 选项数值稍大但本质相同 |
| 稀有事件无独特奖励 | 没有成就/解锁/永久buff等特殊回报 |
| 事件无关联 | 前后事件无因果链，选择不影响后续走向 |
| 恶搞选项无记忆 | 选了搞笑选项后叙事不回顾 |

### 1.4 缺乏新手引导

新玩家面对空白乐园，不知道该先做什么，设施功能和游戏目标不明确。

## 二、优化方案

### 2.1 数值平衡 — 指数级成本增长策略

**核心原则：** 开局不动，保证"建小窝→收养→投放玩耍设施"流程顺畅。通过指数级拉高后续成本制造中后期压力。

参考经典放置/经营游戏的成本模型：每一"层"设施成本是上一层的 2-3 倍，形成 S 型体验曲线（前期引导→中期抉择→后期挑战）。

#### 2.1.1 设施成本重构（指数增长）

**开局不变：** 初始能源 50，木屑小窝 20 + 跑轮 30 = 50，刚好完成基础循环。

**中后期成本拉高：** 按"tier"分层，每 tier 成本倍增：

| Tier | 设施 | 现成本 | 新成本 | 倍率说明 |
|------|------|--------|--------|----------|
| T1 | 木屑小窝 | ⚡20 | ⚡20 | 不变，开局必需 |
| T1 | 跑轮发电站 | ⚡30 | ⚡30 | 不变，开局必需 |
| T2 | 迷宫隧道 | ⚡50 | ⚡60 | 第二个play设施需要积攒 |
| T2 | 食堂 | ⚡50 | ⚡55 | 功能性设施略低于同tier |
| T2 | 向日葵储能站 | ⚡40 | ⚡50 | 储能站适度提价 |
| T2 | 花园别墅 | ⚡60 | ⚡70 | 升级住所有成本 |
| T3 | 滑梯乐园 | ⚡60 | ⚡120 | **关键跳跃**，需天使Lv.2 |
| T3 | 棉花糖泳池 | ⚡80 | ⚡150 | 大容量设施高成本 |
| T3 | 瞭望台 | ⚡70 | ⚡100 ✨5 | 引入星尘消耗 |
| T4 | 云朵城堡 | ⚡120 ✨15 | ⚡250 ✨20 | 后期豪华住所 |
| T4 | 星空蹦床 | ⚡100 ✨10 | ⚡300 ✨15 | 顶级play设施 |
| T4 | 星尘祭坛 | ⚡150 ✨20 | ⚡350 ✨30 | 星尘生产核心 |

**升级成本倍率也随之调高：**

| 设施 Tier | 现倍率 | 新倍率 | Lv.2 升级大致成本 |
|-----------|--------|--------|-------------------|
| T1 | 1.5-1.8x | 2.0x | 木屑小窝 ⚡40，跑轮 ⚡60 |
| T2 | 1.8x | 2.2x | 迷宫 ⚡132，花园 ⚡154 |
| T3 | 1.8-2.0x | 2.5x | 滑梯 ⚡300，棉花糖 ⚡375 |
| T4 | 2.0x | 3.0x | 蹦床 ⚡900，祭坛 ⚡1050 |

#### 2.1.2 引入设施维护费

每回合运营的设施消耗少量能源。无天使管理的设施不运作也不消耗（闲置不罚，但也无产出）。

| 设施 Tier | 维护费/回合 |
|-----------|-------------|
| T1 | 1 |
| T2 | 3 |
| T3 | 5 |
| T4 | 8 |

**效果：**
- 前期(T1+T2)维护费微乎其微(4/回合)，不影响开局体验
- 中期扩建到5-6个设施后维护费开始可感知(15-20/回合)
- 后期全设施维护费可达40+/回合，产能必须跟上
- 4个天使管4个设施刚好，超出开始需要取舍

#### 2.1.3 产能小幅下调

保持前期体验，降低中后期设施的单位产出效率（配合高成本，让升级更有价值）：

| 设施 | 现 basePower | 新 basePower | 说明 |
|------|-------------|-------------|------|
| 跑轮发电站 | 10 | 10 | 不变，开局核心 |
| 迷宫隧道 | 8 | 8 | 不变 |
| 滑梯乐园 | 12 | 10 | 高成本对应略降 |
| 棉花糖泳池 | 6 | 5 | 容量优势补偿 |
| 星空蹦床 | 15 | 14 | 顶级小幅降 |

#### 2.1.4 调整体力节奏

| 参数 | 现值 | 建议值 | 效果 |
|------|------|--------|------|
| 体力消耗/回合 | 15 | 20 | 连续玩耍3回合后需休息 |
| 体力恢复/回合 | 20 | 15 | 需2回合恢复，产生轮换需求 |
| 食堂加成 | +50% | +60% | 食堂更有价值（15×1.6=24） |
| 低体力阈值 | 20 | 25 | 更早自动停工 |

**效果：** 一只鼠鼠工作3回合 → 休息2回合。要维持一个2容量设施持续运转，需要至少3只鼠鼠轮换，鼠鼠数量变得有意义。

#### 2.1.5 星尘经济调整

| 调整项 | 现值 | 新值 | 理由 |
|--------|------|------|------|
| 天使 Lv.2 成本 | 10✨ | 15✨ | 不让初期成就直接升满 |
| 天使 Lv.3 成本 | 25✨ | 40✨ | 对应 T3 设施解锁 |
| 天使 Lv.4 成本 | 50✨ | 80✨ | 后期目标 |
| 天使 Lv.5 成本 | 100✨ | 150✨ | 终极目标 |
| 星尘祭坛产出 | 2✨/回合 | 2✨/回合 | 不变，已够慢 |
| 新增星尘消耗 | — | 事件选项/设施解锁 | 增加花费途径 |

成就奖励保持不变（共85✨），但天使总升级成本从 185✨ 升至 285✨，需要更长期的规划。

### 2.2 天使系统增强

#### 2.2.1 天使管理加成

天使等级影响其管理的设施效率：

```
管理加成 = 1 + (angel.level - 1) × 0.1
```

| 天使等级 | 管理加成 | 实际效果 |
|---------|---------|---------|
| Lv.1 | ×1.0 | 基准 |
| Lv.2 | ×1.1 | 产能/心情恢复 +10% |
| Lv.3 | ×1.2 | +20% |
| Lv.4 | ×1.3 | +30% |
| Lv.5 | ×1.4 | +40% |

纳入产能公式：`(facility.basePower + hamster.basePower) × levelMult × angelMult × personalityBonus`

**效果：** 天使升级有持续可感知的收益回报，不仅仅是解锁高级设施。

#### 2.2.2 天使被动能力

每位天使管理设施时触发独特被动：

| 天使 | 被动名 | 效果 |
|------|--------|------|
| 棉花 | 温馨氛围 | 管理的 living 设施心情恢复 +30% |
| 螺丝 | 精密调校 | 管理的 play 设施产能 +15%，但鼠鼠体力消耗 +3 |
| 星星 | 星辰共鸣 | 管理的设施每3回合额外产出 1✨ |
| 泡芙 | 幸运光环 | 管理的设施中鼠鼠偏好匹配加成从 +50% → +75% |

#### 2.2.3 完善技能实现

**螺丝 - 紧急维修（emergency_repair）：**
- 现状：stub
- 新效果：选定一个设施，立即移除"设施故障"buff（配合 2.3 的 buff 系统）
- 无故障时：该设施下回合产出 ×1.5

**泡芙 - 幸运预感（lucky_hunch）：**
- 现状：stub
- 新效果：使用后在 UI 显示下回合事件类型提示（如"泡芙的耳朵抖了抖：'我闻到了机遇的味道！'"）
- 实现：根据事件概率预计算，写入临时 state 字段

**泡芙 - 好运连连（lucky_streak）：**
- 现状：stub
- 新效果：下回合强制生成至少一个 opportunity 类事件
- 实现：在事件指令 EJS 模板中检查 buff，有该 buff 时指示 AI

#### 2.2.4 泡芙社交域扩展

新增设施 — **交谊厅（lounge）**：
- 类别：function
- manageDomain：social
- 成本：⚡65（T2档位）
- 效果：每回合所有鼠鼠心情 +2（全局微量buff）
- 泡芙管理加成：心情 +3 且随机一只鼠鼠获得"社交活力"buff（下回合产能 +25%）
- requiredAngelLevel: 1

### 2.3 事件系统深化

#### 2.3.1 Buff 系统

在 GameState 中增加 `buffs` 字段，支持持续性效果：

```typescript
interface Buff {
  id: string;           // buff 标识
  type: string;         // 效果类型
  target?: string;      // 目标（hamster_id / facility_id / 'global'）
  value: number;        // 效果数值
  duration: number;     // 剩余回合数
  description: string;  // 显示文本
}
```

事件选项新增可选字段：
```typescript
interface EventOption {
  // ... 现有字段
  buff?: {
    type: string;
    target?: string;
    value: number;
    duration: number;
    description: string;
  };
}
```

**预设 buff 类型：**

| type | 效果 | 典型触发 |
|------|------|----------|
| production_boost | 目标鼠鼠/设施产能 ×(1+value/100) | opportunity 事件正面选项 |
| mood_regen | 目标鼠鼠/全局 心情 +value/回合 | daily 事件、棉花技能 |
| mood_drain | 目标鼠鼠 心情 -value/回合 | challenge 事件负面后果 |
| facility_down | 目标设施产能 ×0.5 | challenge 事件（设施故障） |
| stardust_bonus | 全局星尘收入 +value/回合 | special 事件 |
| stamina_save | 目标鼠鼠体力消耗 -value | opportunity 事件 |

**结算时机：** 在 `settleTurn()` 开始时遍历 buffs，应用效果，`duration--`，到期移除。

#### 2.3.2 挑战事件差异化

**提示词层面：** 在事件指令中为 challenge 类型增加专用规则：

```
挑战事件要求:
  - 必须包含至少一个"冒险选项"（高收益 + 可能触发负面buff）
  - 必须包含至少一个"稳妥选项"（小正收益，无负面效果）
  - 冒险选项的收益应为稳妥选项的 2-3 倍
  - 负面后果以 buff 形式体现，持续 2-3 回合，不直接大额扣资源
  - 负面 buff 可被天使技能提前移除（螺丝修设施、棉花安抚心情）
```

**代码层面：** 事件选择时若选项包含 buff，自动应用到 GameState.buffs。

#### 2.3.3 稀有事件特殊化

special 事件（5%概率）在提示词中明确要求独特奖励：

| 稀有事件模板 | 奖励 | 提示词指引 |
|-------------|------|-----------|
| 天使觉醒 | 指定天使获得临时全技能冷却重置 | 叙事描述天使短暂觉醒 |
| 星尘雨 | stardust_delta: 5-10 | 远超日常的一次性收获 |
| 流浪鼠鼠 | 高 basePower(7-8) 的稀有鼠鼠 | 配合收养系统 |
| 双倍丰收 | 全局 production_boost buff 3回合 | 全设施产能 +50% |
| 天使默契 | 两位天使相关的联合叙事 + 双 buff | 深化天使关系 |

#### 2.3.4 事件链系统

在 GameState 增加 `event_flags: Record<string, number>` （value = 设置时的 turn 号）：

**机制：**
1. 事件选项可包含 `flag_set: "flag_name"` 字段
2. 选择该选项时，写入 `event_flags[flag_name] = current_turn`
3. 事件指令 EJS 模板检查 flags，有 flag 时指示 AI 生成后续事件
4. 后续事件处理后清除 flag
5. 超过 3 回合未触发的 flag 自动过期

**示例事件链：**
```
回合 N（daily）: 星星感应到地下异常光芒
  选择"派鼠鼠探索" → flag_set: underground_glow
  选择"暂时观望" → 无后续

回合 N+1（有 flag 时触发 opportunity）: 发现水晶洞穴
  选择"开采水晶" → stardust +5, 解锁成就
  选择"保护洞穴" → 全局 mood_regen buff 3回合
```

### 2.5 隐藏天使 "???" 解锁

**解锁条件（全部满足）：**
1. 拥有 ≥5 只鼠鼠
2. 至少1位天使达到 Lv.3
3. 回合数 ≥ 20

**触发方式：**
- 满足条件后，special 事件中出现"第五天使降临"剧情
- 该天使 manageDomain = 'special'，可管理任意设施（但管理加成仅为专职天使的70%）
- 解锁新的成就 + 叙事彩蛋

## 三、数值模拟验证

### 优化后前15回合模拟

```
Turn 0: ⚡50/100, ✨0, 💛70
  → 建木屑小窝(-20) + 建跑轮(-30) = ⚡0
  → 收养第一只鼠鼠（免费），分配住所+玩耍
  → 成就"初心" → +5✨

Turn 1: ⚡0 + 产出~15 - 维护2 = ⚡13
  → 鼠鼠体力 80（-20），继续玩耍
  → 事件可能 +5~10 能源

Turn 2: ⚡~20, 鼠鼠体力60
  → 攒钱中，考虑下一步方向

Turn 3: ⚡~33, 鼠鼠体力40
  → 鼠鼠接近疲劳，回窝休息

Turn 4: ⚡~33-2维护 = ~31, 鼠鼠休息恢复体力(+15=55)
  → 无产出回合，能源仅靠事件
  → 可见到第二只收养候选

Turn 5: ⚡~30, 鼠鼠体力70 → 重新玩耍
  → 攒够建食堂(55)或储能站(50)？还差一点
  → 开始面临选择：收养第二只（需第二个住所）vs 建功能设施

Turn 6-8: 逐步积累，收养第二只鼠鼠
  → 两只鼠鼠轮换跑轮，产出更稳定
  → ⚡ 积累到 60-70，可考虑 T2 设施

Turn 9-10: 建第二个设施（迷宫60 或 花园别墅70）
  → 天使升级需求出现（15✨ for Lv.2）
  → 有意义的资源分配决策

Turn 11-15: 进入 T3 准备期
  → T3 设施成本 120+，需要多回合积攒
  → 维护费开始可感知（3-4个设施，~10/回合）
  → buff/debuff 事件增加策略深度
```

## 五、设计原则

1. **开局体验优先**：前3回合必须流畅，"建窝→收养→玩耍→发电"闭环不被打断
2. **指数而非线性**：成本增长用指数曲线，前期感觉不到，中期开始有压力，后期成为驱动力
3. **治愈基调不变**：负面效果温和（buff 而非直接扣除），挑战是"有趣的难题"而非惩罚
4. **AI 提示词同步**：数值改动后，事件指令中的 delta 建议范围要同步调整
5. **渐进式复杂度**：buff/事件链/天使被动等机制在中期自然引入，不在开局堆砌
6. **恶搞选项永远安全**：即使引入风险机制，恶搞选项依然零风险、纯开心
