## 游戏基本逻辑

概括：AI控制随机事件，代码控制变量系统和前端交互

### 核心循环

```
收养鼠鼠 → 鼠鼠在设施中玩耍 → 产生能源 → 用能源建造/升级设施 → 吸引/容纳更多鼠鼠
```

这是一个正反馈循环。玩家的核心决策是**资源分配**：把有限的能源投入到哪些设施、收养哪些鼠鼠、如何应对随机事件。

### 回合制结构

每次玩家发送消息视为"推进一个回合"。一个回合内：
1. **玩家行动**：通过前端按钮或文字指令执行操作（建造、收养、分配等）
2. **结算**：代码自动计算能源产出、消耗、状态变化
3. **AI叙事**：AI 描述本回合发生的事情，包括鼠鼠的日常、随机事件、鼠天使的反馈

## 资源系统

### 能源（⚡ 主货币）

- **来源**：鼠鼠在设施中玩耍产生
- **用途**：建造设施、升级设施、收养新鼠鼠、应对事件
- **存储上限**：由储能设施决定，初始上限较低，鼓励玩家及时使用

### 星尘（✨ 稀有货币）

- **来源**：鼠天使执行特殊任务获得、稀有事件奖励、成就解锁
- **用途**：解锁高级设施蓝图、鼠天使技能升级、触发特殊事件
- **设计意图**：控制游戏节奏，让玩家有长期追求目标

### 心情值（💛 全局状态）

- **含义**：乐园整体幸福度，由所有鼠鼠的平均心情决定
- **影响**：心情越高，能源产出效率越高（乘数加成）；极低时鼠鼠可能"离家出走"（临时不产出，AI 描述它躲起来了，需要鼠天使去安抚）
- **变动因素**：设施品质、拥挤度、随机事件、玩家选择

## 角色系统

### 鼠居民（随机生成）

每只鼠鼠是一个数据实体，由 AI 在事件中随机生成，拥有以下属性：

| 属性 | 说明 |
|------|------|
| 名字 | 由 AI 生成或玩家命名 |
| 品种 | 金丝熊、侏儒、奶茶、布丁、银狐、紫仓等（影响外观描述） |
| 性格标签 | 1-2个，如"贪吃""胆小""好奇""社牛""爱睡觉"（影响 AI 叙事风格和事件触发） |
| 心情 | 0-100，影响产能 |
| 体力 | 0-100，玩耍消耗体力，休息恢复；体力低时自动回窝休息 |
| 玩耍偏好 | 偏好的设施类型，在偏好设施中产能+50% |
| 产能基数 | 每回合基础能源产出 |
| 背景故事 | AI 生成的收养背景（为什么被遗弃等），1-2句话 |
| **记忆** | 该鼠鼠的个体互动记录（见"个体记忆系统"章节） |

**收养机制**：
- 每隔几个回合，AI 会描述"乐园门口发现了一只被遗弃的鼠鼠"，附带品种、性格描述
- 玩家选择是否收养（收养需要消耗少量能源准备住所）
- 乐园容量由住所设施决定，满员时需要先扩建

### 鼠天使（预设角色）

鼠天使是预设的固定角色，不会随机生成。每位鼠天使有独特的人设、性格和能力方向，是 AI 角色扮演的主体。

**与鼠居民的核心区别**：
- 鼠居民是随机生成的、数量增长的"被照顾者"
- 鼠天使是预设的、数量固定的"管理者"，负责运营设施

**基础设定**：
- 外表各异，每位有独特的超自然特征（光环、翅膀、星纹等）
- 每位有完整的预设人格（温柔型、毒舌型、呆萌型、学者型等）
- 有等级系统，等级与设施等级互为条件（见下方）
- 每位有个体记忆，记录玩家与该天使的所有互动历史

**鼠天使列表（预设）**：

| 天使 | 性格关键词 | 管理方向 | 初始技能 |
|------|-----------|---------|---------|
| 棉花 | 温柔、治愈 | 生活设施 | 治愈之触 |
| 螺丝 | 暴躁、认真 | 产能设施 | 能源过载 |
| 星星 | 神秘、安静 | 星尘相关 | 星尘感应 |
| 泡芙 | 话痨、乐天 | 社交/事件 | 幸运预感 |
| ??? | 解锁条件：后期 | 特殊 | ??? |

（具体天使数量和人设后续细化，这里是框架示例）

**鼠天使等级系统**：

天使有等级（Lv.1 → Lv.5），升级需要星尘 + 达到条件：

| 等级 | 解锁能力 | 条件 |
|------|---------|------|
| Lv.1 | 管理基础设施，基础技能 | 初始 |
| Lv.2 | 管理中级设施（如花园别墅），技能强化 | 管理设施累计运行20回合 |
| Lv.3 | 管理高级设施（如云朵城堡），解锁第二技能 | 需要至少1个Lv.2设施 |
| Lv.4 | 解锁被动光环效果 | 需要至少2个Lv.3设施 |
| Lv.5 | 终极形态，特殊剧情 | 需要全部设施Lv.3+ |

**天使 ↔ 设施联动**：

这是核心机制 — 天使等级和设施等级互为前提：
- **设施需要对应等级的天使才能运行**：Lv.2花园别墅需要Lv.2+的生活方向天使（棉花）来管理
- **天使升级需要管理过足够等级的设施**：棉花要升Lv.3，需要乐园已有Lv.2的生活设施
- 这形成了一个交替推进的升级路径：升天使 → 解锁设施 → 运行设施 → 满足下次升天使条件

技能由玩家通过前端按钮触发，效果由代码结算，AI 负责以该天使的语气描述施放过程。

## 随机事件系统

随机事件是每回合的**核心 RP 内容**。每回合 AI 生成 2-3 个事件，玩家对每个事件做出选择，选择直接影响三种资源的增减。这是玩家与 AI 互动的主要方式。

### 事件结构

每个事件由 AI 生成，包含：
1. **叙事描述**：发生了什么（AI 自由发挥，结合鼠鼠性格和乐园状态）
2. **2-3个选项**：每个选项标注资源影响，玩家选择
3. **结果叙事**：AI 根据玩家选择描述后续

**选项设计原则**：
- 至少一个"稳妥正收益"选项（少量但安全）
- 一个"高风险高回报"选项（可能大赚也可能小亏）
- 可以有一个"恶搞/摆烂"选项（小幅倒扣资源，但 AI 会生成搞笑叙事作为"回报"）
- **负面选项是恶搞性质的**，不会真正毁掉游戏进度，扣的量要远小于正常收益

### 事件示例

**事件：布丁在跑轮上跑太快飞出去了**
- 🅰 赶紧去接住她（⚡+5, 💛+10 布丁）
- 🅱 让她自己体验飞翔的感觉（⚡+12, 💛-5 布丁，但布丁解锁"飞天"趣味标签）
- 🅲 你也一起上去转（⚡-3, 💛+15 全体，AI 描写玩家被甩飞的搞笑场面）

**事件：星星天使说她感应到地下有什么东西在发光**
- 🅰 派螺丝去挖掘（✨+8，需要螺丝空闲）
- 🅱 全员寻宝大会！（✨+3, 💛+10 全体，但⚡-10）
- 🅲 假装没听到（无消耗，无收益，但星星会记住你忽视了她 → 写入星星的个体记忆）

**事件：一只流浪猫在乐园门口徘徊**
- 🅰 启动守护结界驱赶（需棉花技能，💛不变）
- 🅱 用食物引开它（⚡-8, 安全解决）
- 🅲 让鼠鼠们表演"装死"吓跑猫（⚡-2, 💛-5，但 AI 描写一段爆笑的装死表演）

### 事件与个体关联

事件可以关联到**具体的鼠鼠或鼠天使**：
- AI 生成事件时会指定关联角色（如"布丁今天发现了个洞"）
- 玩家的选择和结果会写入该角色的个体记忆
- 下次涉及同一角色的事件时，AI 可参考该记忆，形成连续性叙事
- 例：上次布丁飞出跑轮时玩家选了🅱，下次布丁相关事件 AI 可能提到"自从上次飞天体验后，布丁对高处产生了奇怪的执念……"

### 事件分类与概率

每回合生成事件数量：2-3个（代码随机决定）

| 类型 | 每个槽位的概率 | 特征 |
|------|--------------|------|
| 日常 | 50% | 轻松小事，资源波动小（±5~15），推进鼠鼠日常 |
| 机遇 | 30% | 需要决策，资源波动中（±10~30），可能涉及交易/发现 |
| 挑战 | 15% | 需要应对，有小幅负面风险，但"恶搞选项"让失败也有趣 |
| 特殊 | 5% | 里程碑事件，大幅资源变动，可能推进鼠天使剧情 |

### AI 生成事件的约束（提示词侧）

提示词需要告诉 AI：
- 当前资源状况（避免扣到负数的选项）
- 可关联的角色列表和简要状态
- 本回合需要生成几个事件（代码决定）
- 每个选项必须标注 `⚡±X / ✨±X / 💛±X` 格式的资源影响
- 恶搞选项的扣除量不超过当前资源的10%

## 个体记忆系统

核心设计：**每只鼠鼠和每位鼠天使都有独立的记忆变量，只在互动到该角色时才注入提示词**，以节省 token。

### 设计动机

- SillyTavern 的提示词空间有限，不可能每回合把所有角色的历史都塞进去
- 但玩家会期望"上次和布丁的互动被记住了"
- 解决方案：每个角色一个独立记忆槽（存在 MVU 变量中），按需加载

### 记忆结构

每个角色（鼠居民 + 鼠天使）有一个 `memory` 字段，是一个字符串数组，存储该角色相关的互动摘要：

```
角色记忆 = [
  "[回合3] 收养了布丁，她因为主人搬家被遗弃，性格胆小",
  "[回合7] 布丁从跑轮上飞出去了，玩家选择让她体验飞翔，布丁获得'飞天'标签",
  "[回合15] 玩家专门去安慰了受惊的布丁，心情恢复到90",
  "[回合22] 布丁和奶茶成为了好朋友"
]
```

### 记忆的写入时机

以下情况会向角色记忆追加条目：
1. **收养时**：记录收养背景
2. **该角色相关的事件发生时**：无论是主动触发还是随机事件涉及
3. **玩家主动与该角色互动时**：玩家选择"和布丁玩"或"找星星聊天"
4. **重大状态变化**：心情跌破/恢复阈值、获得新标签等

### 记忆的读取时机

以下情况会将角色记忆注入 AI 提示词：
1. **该角色出现在本回合事件中**：AI 参考记忆生成连续性叙事
2. **玩家主动选择与该角色互动**
3. **鼠天使的技能涉及该角色**（如棉花治愈布丁时，注入布丁的记忆）

**不注入的情况**：该角色本回合没有被涉及 → 记忆不占用提示词空间

### 记忆管理

- 每个角色记忆上限：约10-15条（防止变量过大）
- 超出时，由代码自动压缩旧记忆（合并相邻的低重要性条目）
- 标记为"重要"的条目（收养、里程碑）不会被压缩
- 鼠天使的记忆上限更高（~20条），因为她们是主要 RP 对象

### 玩家主动互动

玩家可以在前端选择"与某角色互动"：
- 选择一只鼠鼠 → AI 生成一段以该鼠鼠为中心的小场景（参考该鼠的记忆和性格）
- 选择一位鼠天使 → AI 以该天使的人格进行对话式 RP（参考该天使的记忆和性格）
- 互动结果可能产生小幅资源变动（💛为主）
- 互动内容写入该角色的记忆

这个机制让玩家可以和喜欢的角色建立"关系线"，而不用担心 token 爆炸。

```typescript
// 游戏全局状态
interface GameState {
  energy: number;           // 当前能源
  energyCap: number;        // 能源上限
  stardust: number;         // 星尘
  turn: number;             // 当前回合数
  happiness: number;        // 全局心情值 (0-100)

// 鼠居民（随机生成）

  assignedFacility: string | null; // 当前管理的设施ID
  skills: SkillState[];
  memory: MemoryEntry[];    // 个体记忆（上限更高）
  exp: number;              // 经验值（管理设施累计回合等）
}

// 个体记忆条目
interface MemoryEntry {
  turn: number;             // 发生的回合
  text: string;             // 摘要文本
  important: boolean;       // 重要标记（不会被自动压缩）
}

// 设施
interface Facility {
  id: string;
  type: string;             // 设施类型ID
  level: number;            // 等级 1-4
  capacity: number;         // 容量
  occupants: string[];      // 当前使用的鼠鼠ID列表
  managedBy: string | null; // 管理该设施的天使ID
  requiredAngelLevel: number; // 需要的天使等级
}

// 技能状态
interface SkillState {
  skillId: string;
  level: number;            // 技能等级
  cooldownLeft: number;     // 剩余冷却回合数
  unlockedAtAngelLevel: number; // 天使几级解锁此技能
}

// 本回合事件（AI生成，代码解析）
interface GameEvent {
  id: string;
  description: string;      // AI 生成的叙事描述
  relatedCharacters: string[]; // 关联的角色ID（鼠鼠或天使）
  options: EventOption[];
}

interface EventOption {
  label: string;            // 选项文字
  energyDelta: number;      // ⚡ 变化
  stardustDelta: number;    // ✨ 变化
  moodDelta: number;        // 💛 变化（全局或指定角色）
  moodTarget?: string;      // 心情变化的目标角色ID，空则全局
  isSilly?: boolean;        // 是否恶搞选项
}
```

## 前端界面规划

在 iframe 有限空间内，采用**标签页切换**的布局：

### 标签页结构

1. **总览页**（默认）
   - 乐园名称、回合数
   - 资源栏：⚡能源 / ✨星尘 / 💛心情
   - 本回合产能预览
   - 快捷操作按钮（推进回合等）

2. **鼠鼠页**
   - 鼠鼠列表卡片（名字、品种、心情条、体力条、当前位置）
   - 点击展开详情（性格、背景故事、偏好）
   - 分配到设施的操作

3. **设施页**
   - 已建造设施列表（类型、等级、容量、当前使用者）
   - 建造新设施按钮 → 弹出可建造列表
   - 升级按钮

4. **鼠天使页**
   - 天使卡片列表（名字、等级、管理方向、当前管理的设施）
   - 技能列表（名称、效果、冷却状态）
   - 使用技能 / 与天使互动按钮
   - 升级按钮（显示升级条件是否满足）

5. **事件页**（每回合核心交互区）
   - 本回合的 2-3 个事件卡片
   - 每个事件显示叙事描述 + 选项按钮（标注资源影响）
   - 选择后显示结果叙事
   - 恶搞选项可用特殊样式标注（如斜体/删除线暗示"你确定？"）

6. **日志页**
   - 近期事件记录（简略版，详细版由AI叙事）
   - 成就列表与进度

## AI 与代码的分工

| 职责 | 代码 | AI |
|------|------|-----|
| 资源计算 | ✅ 精确结算 | |
| 设施建造/升级 | ✅ 数值变更、前提检查 | 描述建造过程 |
| 每回合事件生成 | 决定数量和类型槽位 | ✅ 生成叙事、选项、资源标注 |
| 玩家选择的后果 | ✅ 数值结算 | ✅ 生成结果叙事 |
| 新鼠鼠生成 | 确定属性范围 | ✅ 生成名字、性格、故事 |
| 个体互动 | 加载/保存个体记忆 | ✅ 参考记忆进行 RP |
| 鼠天使对话 | 技能冷却和效果 | ✅ 以该天使人格进行 RP |
| 记忆管理 | ✅ 写入、压缩、按需注入 | 生成记忆摘要文本 |

### AI 提示词注入策略

**每回合必定注入**：
- 当前资源状况（⚡/✨/💛 + 上限）
- 回合数、鼠鼠总数
- 本回合需生成的事件数量和类型槽位
- 所有鼠鼠的名字+品种+性格标签（简表，不含记忆）
- 所有鼠天使的名字+等级+状态

**按需注入（仅涉及到时）**：
- 被关联角色的个体记忆（事件涉及该角色时）
- 玩家主动互动的角色的个体记忆
- 鼠天使技能描述（使用技能时）

这样设计使得基础提示词保持精简，只在需要时加载详细记忆。
